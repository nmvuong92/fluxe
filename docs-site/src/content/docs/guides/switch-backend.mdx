Cell `todos` chỉ gọi `backend.addTodo(title)`. **Backend nào thực thi** là do Resolution Plane
giải — bạn không sửa cell. Dưới đây là *cùng một hợp đồng* `addTodo(title) → Todo` ở **7 ngôn ngữ/runtime**.

<Aside type="caution" title="Đừng hiểu nhầm: HTTP-swap KHÔNG phải tính năng tốc độ">
Mặc định là **`memory` in-process — 0 roundtrip** (gọi hàm cùng tiến trình). Đẩy qua HTTP sang
Go/Rust **tốn roundtrip + serialize**, với việc nhẹ là *chậm hơn*, không nhanh hơn. Chỉ resolve
sang service riêng khi **đo được** lợi: compute nặng (Rust hơn hẳn dù trừ overhead), hoặc cần tách
ops (scale/deploy/đội ngôn ngữ độc lập) — lúc đó bạn trả roundtrip *dù dùng kiến trúc nào*.

Điều "đặc biệt" là **decoupling**: start in-process, chỉ phân tán khi đáng — **không viết lại cell**.
Transport cũng không bắt buộc HTTP (in-process / Unix socket / gRPC… đều là adapter); HTTP ở đây chỉ
là lựa chọn demo đơn giản.
</Aside>

Tất cả expose cùng HTTP: `GET /todos` · `POST /todos {title}` · `POST /todos/{id}/toggle`,
và gắn prefix (`[Go]`, `[Rust]`, `[Python]`…) để bạn thấy rõ request đi qua service thật.

## Đổi backend = đổi profile

Không có lệnh nào trong cell. Chỉ một dòng trong `app/profiles.ts`:

```ts
// app/profiles.ts
export const profiles: Record<string, ResolutionProfile> = {
  dev:        { name: "dev", backend: "memory" },
  "prod-go":  { name: "prod-go", backend: "go",   endpoints: { go: "http://127.0.0.1:8081" } },
  "prod-rust":{ name: "prod-rust", backend: "rust", endpoints: { rust: "http://127.0.0.1:8082" } },
  // Per-cell: app mặc định memory, riêng cell `todos` (hot) giải sang Go
  mixed:      { name: "mixed", backend: "memory",
                endpoints: { go: "http://127.0.0.1:8081" },
                cellBackends: { todos: "go" } },
};
```

## Cổng & lệnh chạy mỗi service

| Backend | Port | Lệnh chạy service |
|---------|------|-------------------|
| memory  | —    | (in-process, mặc định `npm run dev`) |
| Go      | 8081 | `go run ./app/native/go` |
| Rust    | 8082 | `rustc -O app/native/rust/main.rs -o server && PORT=8082 ./server` |
| Python  | 8083 | `PORT=8083 python3 app/native/python/server.py` |
| Hono    | 8084 | `cd app/native/hono && npm i && PORT=8084 npm start` |
| .NET    | 8085 | `PORT=8085 dotnet run --project app/native/dotnet` |
| Java    | 8086 | `PORT=8086 java app/native/java/Server.java` |

## Hai cách switch

<Tabs syncKey="run">
  <TabItem label="Live swap (devtools)">
Bật service muốn dùng, rồi mở **DebugBar** (nút ⚡) → chọn backend. Không restart fluxe.

```bash
PORT=8083 python3 app/native/python/server.py   # 1. bật Python
npm run dev                                      # 2. fluxe vẫn memory
# 3. DebugBar → backend: python  →  todo mới có prefix [Python]
```
  </TabItem>
  <TabItem label="Profile (config)">
Cố định backend qua `FLUXE_PROFILE` — không cần devtools.

```bash
go run ./app/native/go
FLUXE_PROFILE=prod-go npm run dev   # mọi action todos đi qua Go
```
  </TabItem>
</Tabs>

:::note[Engine không phân biệt ngôn ngữ]
`devBackend()` map tên ngôn ngữ → URL (override bằng `<LANG>_URL`). Vì hợp đồng là HTTP
thuần, thêm một runtime mới = thêm 1 service expose `/todos` + 1 dòng vào map. Engine,
cell, frontend **không đổi**.
:::

:::tip[Live swap trong devtools]
Không cần restart: mở **DebugBar** (nút ⚡ góc dưới-phải), chọn backend `go`/`rust`.
Response trả header `x-fluxe-resolution: rust/http (swap)` và todo mới có prefix `[Rust]` —
bằng chứng request đi qua service thật.
:::
