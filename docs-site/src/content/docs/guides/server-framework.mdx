import { Tabs, TabItem, Aside } from '@astrojs/starlight/components';

fluxe **không bắt bạn dùng server của nó**. `app/backend/server.ts` chính là **backend CỦA BẠN** —
một app Express / Fastify mà `fx init` đã **wire sẵn adapter fluxe**. Bạn ghi toàn bộ logic
backend ở đó (route API, middleware, service, auth riêng…); fluxe mount **catch-all** ở cuối, và
**core của nó lo các vấn đề xuyên suốt** cho backend đó:

> cells/SSR · contract (`/__rpc`, `/__action`) · realtime (SSE) · validation (Zod) ·
> observability (`/_fluxe`) — bạn **không phải tự dựng lại**.

<Aside type="caution">
**Auth · CSRF · rate-limit · upload · jobs là việc của HOST.** fluxe không kiểm CSRF/rate-limit
ở `/__rpc`/`/__action` — mount middleware host (`express-rate-limit`, `csurf`, `passport`,
`multer`, `bullmq`…) **TRƯỚC** `fluxe()`. `ctx.session` là do host gắn (`req.session`); fluxe chỉ
đọc, không verify.
</Aside>

Demo mặc định dùng **Express + TypeScript**.

<Aside>
Không muốn framework nào? `makeServer(...)` (node:http, zero-dep) vẫn là đường zero-config —
cùng core, cùng concerns.
</Aside>

## `app/backend/server.ts` — entry của bạn

Mỗi adapter là một subpath riêng; framework là **peerDependency** (chỉ cài cái bạn dùng).

<Tabs syncKey="server">
<TabItem label="Express">

```bash
npm i express
```

```ts
// app/backend/server.ts
import express from "express";
import { readFileSync } from "node:fs";
import { fluxe } from "@nmvuong92/fluxe/express";
import type { ResolutionManifest } from "@nmvuong92/fluxe";
import { cells } from "../app";
import { layouts } from "../layouts/index";
import { backend } from "./data";

const manifest: ResolutionManifest = JSON.parse(readFileSync(".fluxe/resolution.json", "utf8"));
const app = express();

// 👉 Logic backend RIÊNG của bạn (chạy TRƯỚC fluxe) — dùng chung `backend` với cell:
app.get("/api/todos", async (_req, res) => res.json(await backend.listTodos()));

app.use(fluxe(manifest, cells, layouts, { backend }));   // fluxe = catch-all (cells + core concerns)
app.listen(5180);
```

</TabItem>
<TabItem label="Fastify">

```bash
npm i fastify
```

```ts
// app/backend/server.ts
import Fastify from "fastify";
import { readFileSync } from "node:fs";
import { fluxe } from "@nmvuong92/fluxe/fastify";
import type { ResolutionManifest } from "@nmvuong92/fluxe";
import { cells } from "../app";
import { layouts } from "../layouts/index";
import { backend } from "./data";

const manifest: ResolutionManifest = JSON.parse(readFileSync(".fluxe/resolution.json", "utf8"));
const app = Fastify();

// 👉 Route Fastify RIÊNG của bạn (chạy TRƯỚC fluxe, giữ body-parsing của Fastify):
app.get("/api/health", () => ({ ok: true }));

await app.register(fluxe(manifest, cells, layouts, { backend }));   // fluxe = catch-all
await app.listen({ port: 5180 });
```

<Aside type="note">
Adapter dùng hook `onRequest` + `reply.hijack()`: request khớp route Fastify của bạn giữ nguyên
(kể cả body-parsing); phần còn lại (`req.is404`) do fluxe lo trên node raw → SSR stream/SSE nguyên vẹn.
</Aside>

## Cơ chế

Cả hai adapter wrap **chung một** `createHandler(manifest, cells, layouts, opts)` — handler Node
`(req, res)`. Express/Fastify dựng trên `node:http` nên dùng thẳng raw.
`makeServer` chính là `http.createServer(createHandler(...))` — nên mọi cách cho hành vi y hệt.

| Cách chạy | Dùng khi |
|-----------|----------|
| `makeServer(...).listen()` | zero-config, zero-dep, không cần framework |
| `@nmvuong92/fluxe/express` | nhúng vào app Express / muốn middleware Express |
| `@nmvuong92/fluxe/fastify` | app Fastify (perf: ~9% req/s cao hơn Express, đo được; plugin encapsulation) |

## Mount = catch-all

fluxe xử các route của nó (cells, `/__rpc`, `/__action`, `/__sse`, `/_fluxe`…). Đặt
route/middleware **riêng của bạn TRƯỚC** `fluxe()` → chúng được ưu tiên (kể cả auth/CSRF/
rate-limit/upload của host); phần còn lại fluxe nhận. Scaffold nhanh:

```bash
fx init --server express   # hoặc --server fastify (mặc định express)
```
