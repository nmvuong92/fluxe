import { Tabs, TabItem, Aside } from '@astrojs/starlight/components';

fluxe **không bắt bạn dùng server của nó**. `app/backend/server.ts` chính là **backend CỦA BẠN** —
một app Express / Hono / Nest mà `fx init` đã **wire sẵn adapter fluxe**. Bạn ghi toàn bộ logic
backend ở đó (route API, middleware, service, auth riêng…); fluxe mount **catch-all** ở cuối, và
**core của nó lo các vấn đề xuyên suốt** cho backend đó:

> cells/SSR · session/CSRF · rate-limit · realtime (SSE) · validation · upload · observability
> (`/_fluxe`) — bạn **không phải tự dựng lại**.

Demo mặc định dùng **Express + TypeScript**.

<Aside>
Không muốn framework nào? `makeServer(...)` (node:http, zero-dep) vẫn là đường zero-config —
cùng core, cùng concerns.
</Aside>

## `app/backend/server.ts` — entry của bạn

Mỗi adapter là một subpath riêng; framework là **peerDependency** (chỉ cài cái bạn dùng).

<Tabs syncKey="server">
<TabItem label="Express">

```bash
npm i express
```

```ts
// app/backend/server.ts
import express from "express";
import { readFileSync } from "node:fs";
import { fluxe } from "@nmvuong92/fluxe/express";
import type { ResolutionManifest } from "@nmvuong92/fluxe";
import { cells } from "../app";
import { layouts } from "../layouts/index";
import { backend } from "./data";

const manifest: ResolutionManifest = JSON.parse(readFileSync(".fluxe/resolution.json", "utf8"));
const app = express();

// 👉 Logic backend RIÊNG của bạn (chạy TRƯỚC fluxe) — dùng chung `backend` với cell:
app.get("/api/todos", async (_req, res) => res.json(await backend.listTodos()));

app.use(fluxe(manifest, cells, layouts, { backend }));   // fluxe = catch-all (cells + core concerns)
app.listen(5180);
```

</TabItem>
<TabItem label="Hono">

```bash
npm i hono @hono/node-server
```

```ts
// app/backend/server.ts
import { Hono } from "hono";
import { serve } from "@hono/node-server";
import { readFileSync } from "node:fs";
import { fluxe } from "@nmvuong92/fluxe/hono";
import type { ResolutionManifest } from "@nmvuong92/fluxe";
import { cells } from "../app";
import { layouts } from "../layouts/index";
import { backend } from "./data";

const manifest: ResolutionManifest = JSON.parse(readFileSync(".fluxe/resolution.json", "utf8"));
const app = new Hono();

// 👉 Route Hono RIÊNG của bạn (chạy TRƯỚC fluxe):
app.get("/api/health", (c) => c.json({ ok: true }));

app.use("*", fluxe(manifest, cells, layouts, { backend }));
serve({ fetch: app.fetch, port: 5180 });
```

<Aside type="note">
Hono chạy trên Node qua `@hono/node-server`. Adapter buffer response rồi trả Web `Response`
(cần stream SSR? dùng Express hoặc `makeServer`).
</Aside>

npm i @nestjs/common @nestjs/core @nestjs/platform-express reflect-metadata

```ts
// app/backend/server.ts
import "reflect-metadata";
import { Module } from "@nestjs/common";
import { NestFactory } from "@nestjs/core";
import { readFileSync } from "node:fs";
import { fluxeMiddleware } from "@nmvuong92/fluxe/nest";
import type { ResolutionManifest } from "@nmvuong92/fluxe";
import { cells } from "../app";
import { layouts } from "../layouts/index";
import { backend } from "./data";

@Module({ /* controllers/providers riêng của bạn */ })
class AppModule {}

const manifest: ResolutionManifest = JSON.parse(readFileSync(".fluxe/resolution.json", "utf8"));
const app = await NestFactory.create(AppModule);

app.use(fluxeMiddleware(manifest, cells, layouts, { backend }));   // mount global, catch-all
await app.listen(5180);
```

## Cơ chế

Cả ba adapter wrap **chung một** `createHandler(manifest, cells, layouts, opts)` — handler Node
`(req, res)`. Express/Nest dựng trên `node:http` nên dùng thẳng; Hono dùng `@hono/node-server`.
`makeServer` chính là `http.createServer(createHandler(...))` — nên ba cách cho hành vi y hệt.

| Cách chạy | Dùng khi |
|-----------|----------|
| `makeServer(...).listen()` | zero-config, zero-dep, không cần framework |
| `@nmvuong92/fluxe/express` | nhúng vào app Express / muốn middleware Express |
| `@nmvuong92/fluxe/hono` | app Hono |
| `@nmvuong92/fluxe/nest` | app NestJS (module/DI có sẵn) |

## Mount = catch-all

fluxe xử các route của nó (cells, `/__action`, `/__sse`, `/_fluxe`, `/__upload`, `/__file`,
`/login`…). Đặt route/middleware **riêng của bạn TRƯỚC** `fluxe()` → chúng được ưu tiên; phần
còn lại fluxe nhận. Scaffold nhanh:

```bash
fx init --server express   # hoặc --server hono | --server nest (mặc định express)
```
